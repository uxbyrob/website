How it works

The accordion has two main parts, the header containing the title and the inline content that’s revealed when the accordion is activated or opened. The default state of the accordion should be closed. On click of the header, the inline content should reveal visually over a period of not less than 500ms, appearing to “open” up from the clicked header. The accordion content area “pushes down” other elements on the page. The header icon should change to indicate the “open” or revealed state by rotating down.

 

Clicking on the header of an “open” accordion ( including the open/close icon ) will close the accordion. The accordion should close in the reverse motion with which it opened, and at the same speed.

 

Where to use it

The accordion is a progressive-reveal element for text or other elements that is visually offset from other elements to indicate it acts different than surrounding default elements. Its purpose is to present the heading of the container as a cue as to what the contained elements are, and so save space by presenting the title as primary and the elements as secondary. Upon activating, the container reveals its contents, making them primary.

 

The accordion is typically used on FAQ pages to facilitate question browsing, minimize page height by keeping answers secondary. Accordions should not be nested, and each should bring value to the page experience. Accordions should not be used with form fields, in commerce contexts, or any situation where user-entered content should remain scannable.

 

The accordion should not store elements with complex interactivity.
