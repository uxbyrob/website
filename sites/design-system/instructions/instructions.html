Understand IHSM's Internal Nexus Server

A note on registry chaining: We recommend that you point your npm requests at IHS Markit's public-npm-all aggregate registry unless you have a specific reason not to do this. This will allow the registry chaining and built-in fallbacks for the public-npm-all aggregate registry to kick in when dependency packages must come from multiple sources (some internal and some public).

This means that any request for an npm package that this aggregate registry receives will be requested from the following registries in this order:

  1. IHS Markit's public-npm registry
  2. IHS Markit's npm-registry mirror
  3. npm-registry

Requests for the csspatternlibrary3 npm package only work properly when requested from the public-npm-all aggregate registry

Thus, an npm package request for csspatternlibrary3 (which has a dependency of its own on Bootstrap) will be looked up first in IHS Markit's public-npm registry. When csspatternlibray3 is found in IHS Markit's public-npm registry it will be downloaded and its package.json file will be inspected for further dependencies.

The dependency on Bootstrap will be found. Which will then initiate a request for the bootstrap npm package from IHS Markit's public-npm registry which will also follow the chain until it is found. The bootstrap npm package (which is not an IHS Markit specific package and is publicly available to everyone) is not in IHS Markit's public-npm registry. It may not yet be in IHS Markit's npm-registry mirror, but it will definitely be found in the npm-registry. After the bootstrap npm package is downloaded it's package.json file will be inspected for dependencies (finding none) and the chain will stop.

Steps

  1. Assure you are correctly set up to use the IHSM internal npm server.

    You must be connected to the IHSM network (possibly via VPN).

    We expect you will want to set up your computer to use IHSM's internal npm server and services whenever possible. With the correct setup your request for an npm package will first look there for any packages and then if they are not found the request will be forwarded to the web's public npm-registry at https://registry.npmjs.org/
    To do this add a .npmrc (i.e. npm resource) file to your user profile (something like "C:\Users\gxb44509" on Windows) that has the following content: 
    always-auth=true
email=john.doe@ihsmarkit.com
init.author.email=john.doe@ihsmarkit.com
init.author.name=John Doe
init.author.url=http://ihs.com
registry=http://npm-na.ihs.com/repository/public-npm-all/
_auth=SomeCharactersRepresentingA64BitEncodingOfYourUsernameAndPassword

    Gotcha If this file does not already exist you may need to create it at the command line to be able to name it to ".npmrc"
    Gotcha You must encode your username and password into a 64 bit string and place it into the file - replacing "SomeCharactersRepresentingA64BitEncodingOfYourUsernameAndPassword" in the code example above. Sadly, this will need to be updated as you change your password to continue downloading packages from IHS Markit's public-npm-all registry. This can easily be done by using the 64 bit encription utility form we created for this purpose. Alternatively you can also accomplish this via Powershell - see the IHS - Dependency Management - Nexus.docx for instructions on how this is done.


    If, on the otherhand, you simply want to set up a single project folder to use IHSM's internal npm server and services it is very simple to set up.
    Add a .npmrc (i.e. npm resource) file to the root folder (usually the one with the "node_modules" folder in it) of your project that has the following content: 
    always-auth=true
registry=http://npm-na.ihs.com/repository/public-npm-all/
    See the IHS - Dependency Management - Nexus.docx if you encounter difficulties.
  2. Run the install command 
    npm install csspatternlibrary3 --save
     from the command line of your project root folder (or wherever you are currently storing your "node_modules" folder).
  3. Include the needed CPL3 resources in your product.
    The main CSS file (which already includes all needed Twitter Bootstrap styles) 
    <link rel="stylesheet" href="css/styles-nw-min.css" />
    Some components will require the inclusion of their corresponding JavaScript files to function correctly. For example the Carousel component requires 
    <-- Either the bootstrap.min.js file which includes all of the JavaScript for all of the CPL3 components bundled together -->
<script src="js/bootstrap.min.js"></script>

<-- Or the individual carousel.js if it makes more sense to cherry-pick out only what you need. -->
<script src="js/carousel.js"></script>
