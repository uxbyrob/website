US83665 [MAP] - Search For: - Choosing a Search Topic


As a customer in Connect Map who wants to search for a particular location or item,


I can use the Search For: interface to first, select the topic area I would like to search and secondly - to type a search term to search for so that I can quickly navigate geospatially to a location or piece of content in the Map interface.


- user can select a topic area to search

- Topic Area list is defined by map services available to query PLUS Location, which is default

- Location is available to all clients in Connect Map

- Location is default search topic area

- Other topics listed are based on user entitlements and content made available to all users via MapAdmin QUERY functionality for a specific layer

- Topic Areas are grouped by Map Service (i.e.  - all the pipes, facilities, electrical plants layers are grouped under Midstream Essentials)

- Only one layer can be queried at a time.

- Only one group can be maximized at a time (i.e. - you open Midstream, then Chemical Capacity closes)


MAYBE 3.9 - My Locations layers are available to search within

Search term for these layers will always be the location's "NAME" attribute

My Location layers are listed in the search list, grouped under "My Locations"

When a user selects a Topic Area, the Search For: dialogue box changes from reading "Search For: Locations" - which is the default - to "Search for: Fields-Detail (example)"

Search topic area selection persists within a map session (e.g - if user stays in the map interface, or only leaves it to navigate to an article view, and navigates back using the breadcrumb)

Defaults back to Location when user leaves the map interface or starts a new session.

Fe8407 [MAP] - Action Panel -  Definition


As a customer in Connect Map who wants to perform a particular non-geospatial action (i.e. - not related to focusing the map extent) -

I can access an "Action Panel" which is relevant to the tasks or workflow that I want to perform so that I can remain focused, within the interface, on a specific task or workflow - and have maximum usable map space when I no longer am performing said task.

 

Functionality:

- Map interface divides into zones for ease of use: Action Panel, Basic - Navigation, Map, Content Panel* (initially for Data Grid View)

Action Panel will be where user takes actions that will impact the display of information within Connect Map, or means of manipulating that content (tools)

- Action Panel is designed to focus user on the task at hand - for example, displaying oil fields on the map - not on manipulating the map geospatially.

Behavior:

- AP will slide in/out from the left side of the interface - triggered by clicking on one of the Action Panel Icons (see below)*

Initial "Panel" groupings: Layers, Tools, Research/Search, My Connect Map  (each will be described in detail in other features)

Action Panel Icon List - *Accessing Panel:

User will open a specific action panel (such as the Map Layer Menu) - there will be one panel displayed at a time (a single panel could have multiple tabs) - based on user icon selection

"Hamburger" menu option - also opens action panel (???)

Panel States: (opened to a particular panel - e.g. Layers, or closed entirely)

Default state: closed, initially

Within a session - Active State persists - if a user opens the Layer panel and leaves it open, it should persist even if she navigates away from the map, within a session.

*REVERT TO CLOSED WHEN SESSION ENDS (?)

Note that this pertains to the state of the action panel, not the content manipulated within the panels which will be discussed in other features.

Track panel usage through GA event(s): Open, Close and which Panel user is accessing.
