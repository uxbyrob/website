Problem

Solution

Improve notification and process to user for downloading or queuing files for process to export, etc for both immediate and multiple downloads. Chemical business line has an immediate need to make the experience better. Connect has a need for the solution to be leveraged throughout the entire platform. A major issue is users click multiple times on a file that i says ready and shows process complete, thinking that it has already been downloaded on their local machine.


Two items to solve for are immediate and multiple downloads. We would like to recommend removing the file(s) "black" bar at the bottom and move to a "growler/toast" pattern for the right side user notification. When user clicks icon, animate the download icon and note to user that the document is being queued/processed for download, Download can automatically start once complete for an immediate scenario. For a multiple item scenario or long processing document. A growler message is shown on right side indicating that the export is in process and when ready for download. These can stack vertically down the right side of screen for more than one.


Download notification needs to clearly show what is being processed and prepared for download in contrast to what is actually ready for download. If we do stick with the black bar, this needs to not look so native like the browser itself (different color for bar/theme) and must have two sections to indicate processing and ready for download.
