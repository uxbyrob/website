Color uses

Clicks Hover / Focus

The brand-color-primary is used to communicate that an object is clickable; Encouraging a user to interact with the UI. This color is also used to identify the active item in a list group.

It may help to think of it as also being called "brand-color-action" indicating available actions to the user.

For hover / focus states use the dark variation of the appropriate main color.

Go ahead … hover and click the color to the left. It won't hurt.   ;)

brand-color-secondary

The brand-color-secondary color is meant to draw a bit more of the user's attention. It is commonly used as the text color of headings. It should be used judiciously.

Side note brand-color-secondary is technically IHS Markit's primary color from a "branding" perspective. Because web development places great focus on getting users to perform actions we went ahead and swapped the 'primary' and 'secondary' colors (as labeled by branding) for you - making life much easier. Action colors are used much more on the web and this way you won't have to fight Bootstrap's convention of labeling the action color the "primary" color.

brand-color-gradient

brand-gradient is commonly used in the search bar of the header pattern. Just as the brand-secondary color is meant to draw a bit more of the user's attention, the brand-gradient draws just a bit more. It should be used sparingly.

Not Disabled Disabled

Disabled state is 65% opacity and does not specify a color

Intended use is built into color names

For brevity it is assumed that most colors are intended as background colors. Colors that are intended for typography, foreground, border, outlines, or other uses will include their specific intent in their CSS class name. e.g. text

Thus, we have common background colors (like color-success) and special use colors (like brand-color-primary-text). We hope this keeps names as short as possible while allowing developers to quickly sort and find their target's name when assisted by IDE tooling.

brand
Specific to IHS
color
All colors have “color” in their name
bg
Explicitly a background color
text
Specifies a typography / foreground color
Success

Use contextually to indicate success status of user actions.

Info

Use contextually to draw an extra bit of the user's attention to general information, minor changes, and other snippets of information that can safely be glanced over after they've been read the first time.

Warning

Use contextually to indicate warning status of user actions. Also commonly used to indicate that our products detect risk of data loss, unfinished (not-saved) changes, connectivity issues, out-of-synch systems, etc.

Additionally, use it as if it were also called "brand-color-attention" to attract the user's attention when needed.

Danger

Use contextually to indicate error / danger status of user actions when the user should be notified of failures, data loss, road blocks, etc.
