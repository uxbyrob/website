How does the user access the Master List Selector?


- via a link in the drop down menu, (see specs)

- there are cases where the large list selector could be accessed via an icon/or data lozenge/label ..



Are there related expectations of design on the page or in the drop down to account for?


- On focus, drop down menu is shown always

- If user hits enter key in input box, redirects to closest related page or main search results  (TBD)

- Drop down for navigation always show closest match up to 10, recent selections up to 5, browse full list, search within results



Can the selection in the list selector lead to no results in the query?


- No, the list of companies should be filtered by prior user selections

- If possible, grey out list items that are not available, but could be if the parent or grandparent filters were changed, etc.



Is keyword search across all letter/categories  or the current letter/category only?


- across the current filtered items, not categories



How do letters and categories interact?


- categories panel can be displayed or hidden via radio selection.

- User can click category and then letters filter from that selection



Can there be more than one type of category? (We used to have multiple categories, I.E.  Regions and My Regions)


- Yes there can, with option to group by sub-category. Example.. (All) (Regions) (My Stuff > My Regions, My Locations)



Copy for the various parts of the design (i.e. is this "final" copy via UX or for the BL's to decide?)


- Please user recommended verbiage and design provided in the specs. This should cover 95%+ of implementation.

- There will always be unforeseen circumstances that a business line or user might need, that we can address as enhancements



What kind of limited use cases can be applied, only the things that are shown, or only where there is no A-Z? Could we apply this selector without an A-Z list? 


- we could, but not sure why, yet ... this is added feature to the large list selector that helps find names.

- if list is very small, we could look at removing the A-Z, will need to weigh out if important.



Is the same design to be used for navigation and filtering? (how should we handle the selection panel in a navigation use case?)


- yes it is, one would be single links to a landing page,

- the other would be multiple selection to be used in a query, please see examples in document.
