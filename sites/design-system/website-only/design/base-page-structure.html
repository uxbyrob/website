Page Structure

Basic page layout

All apps are different but when possible, basic page structure should be followed.

  • The standard IHS Markit application header is used at the top of the page and contains the main navigation
  • Secondary page navigation or filters go in the left rail. See more information on the navigation page.
  • The center of the page is the primary work area for the user.
  • The far right side of the page is reserved for secondary content and links. This is the lease valuable real estate on the page.
  •  

Remember, the user cares the most about the content in the primary work area and everything else is just a way to navigate or control that information. Always focus on making the largest work area possible.

 

Page height

The system has two page height options:

1) Single page application - Page does not scroll but the content inside containers can scroll.

 

2) Scrolling page - Footer is pushed to the bottom of the content and the page scrolls.

Breakpoints

This system is designed for desktop and tablets and is not optimized for phone screens. The system is designed for 3 screen sizes:

  • Small screens: 1024px X 768px to 1280px X 800px . Screens under 1024X will scroll horizontally.
  • Medium screens: 1281px X 801px to 1600px X 1024px
  • Large screens: 1601 and higher

 

Note: Both the X and Y dimensions mush exceed the breakpoint.

 

At each breakpoint the gutter, font sizes and base spacing between elements change.

 

Design tip: 1366px X 768px falls in the small screen size breakpoint because of its height. Designing in this size represents a typical laptop but can skew your design choices based on its extreme aspect ratio. Consider designing at 1366px X 801px using the medium size elements
