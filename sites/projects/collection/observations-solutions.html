Data Browser Access:

Too many of our user calls repeated this theme over and over, they could not find data browser. Users once figuring out the mega menus did navigate more easily too second and third rounds of usage.

Obervations

Recommended Solutions

Move Data Browser "word" to main navigation bar. This shouldn't have any more precedence then "map" as an example

Refreshable Worksheets Indicator:

Users did not know when their Excel spreadsheet was setup to be refreshable or many times how they set this up in the first place to be refreshable.

Recommend to show on Excel spredsheet and indicator if sheet is refreshable, unless we just by default make all sheets refreshable and there isn't a static state, etc


Also that same indicator in My Connect if the sheet is refreshable, despite it being in this area users don't know it's setup and ready.

Data Series selector and preview:

We saw users continually want to tweak settings or filter their search parameters. Users had to bounce back and forth between search screen and result/preview screen

Design UI to accomodate preview and selection in the same experience. This is key to a user settings up the largest pool of data to export and if user wants to narrow their query seing that more realtime

Data Filter units: (preset)

We had users bring up on our sessions the wish to have a more "smart/dynamic" selection of units of current amounts or weights to be relational to their selections or parent selections

Setup an data matrix that can be ran client side in browser, or either a currency tool/calculator or the filtering.

Export and Go

Users main need and workflow of the data browser is to make their fitler selections and export. Many times they didn't even look at the preview.

Remove and hide table controls, other distracting UI elements that aren't needed as the main or majority of use of the data browser. They can still be accessed, just not visible from default state

Lessen steps to export and design more emphasis on that function and feature. Can we at this stage save in My Connect also at the same time, so refreshable sheet is setup, etc.

UI Table and preview controls

Users just didn't use them, go near them or investigate what their function was in our user sessions. Until pointed out, which for some there was comprehension but no noticable feedback on value of the controls. Orientation was important in one session for displaying of correct format in Excel

Excel Data:

Users want all that can be dumped into these, tables, with individual columns. We heard 90% of our users express frustration with parsing through concatenated columns and the time it takes to seperate out the data.


We also heard from a few that they would like to see aggregated data if presented different then the detail sheets.

Silo all of our table results to a column for each. Strive to get the data segmented as possible, which will allow the user easier setting of pivot tables, report, and parsed information for their needs in their workflow.

Filters and Setup: Searching ...

We experience a few users looking for specific facets in their main filter sections, but were already in a bucket that narrowed that scope. This causes problems if a user is looking for certain country/subject, etc.

Have search not only search the current bucketed results, but have option for user to search more overall or main countires, etc while setting up their search criteria

Data Periods and Record Limits:

Users did want to see from the earliest ot time in our data and did not understand why we limited this amount and also the limitation on records pulled overall in their query amounts

Include historical information in the excel export alongside realtime data. Improve performance to prove larger datasets and record counts.
