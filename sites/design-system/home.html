Important changes give scope to our styles and keep them from leaking October 9th, 2017

Breaking change

csspatternlibrary3 version 2.0.0 is coming

We have introduced an important, but breaking change into the upcoming version of the CPL3.

We hope to release an update to CPL3 on Monday (16 Oct. 2017). Our original target for this release was Friday (13 Oct. 2017) but we've decided to delay so that we can schedule an open "Office Hours Forum" meeting before the release. Please attend the WebEx meeting if you have any concerns.

We're happy to announce that this update will give you more control. It will minimize how often our styles leak out and unintentionally change other styles. To keep with semantic versioning - This breaking change will take effect in our release of the csspatternlibrary3 version 2.0.0 npm package and carry forward from there. The most recent version not effected by this change will be csspatternlibrary3 version 1.0.12

See our roadmap blog post about it for more details. The post is titled "CPL3 scope changes in version 2.0.0".

Please start making the appropriate changes to your project to continue to receive updates from the CPL3. Contact Michael Witt with any questions or concerns you have.

CssPatternLibrary3
design system

Who are we? The IHS Markit (IHSM) Design Systems Team is responsible for this CssPatternLibrary3 design system. We're designers and developers with some big ideas. And we're betting you've got some as well. Please contact Mark Schraad with any questions or feedback you have about our system.

Current version

version 2.0.3

Dependencies

CPL3, in its current version, is mostly based on the Twitter Bootstrap version 3.7. Future versions will attempt to maintain an appropriate measure of backwards compatibility with what has already been established in the CPL3 and the conventions built into Bootstrap. Moving foward our intentions are to do more and reach further than what was ever intended for Bootstrap alone. Thus, we have the current dependencies:

  1. Twitter Bootstrap version 3.7
    1. jQuery

About the design system

Design systems require investment, compliance, and conformity; In return they provide a strong foundation of stability and cohesion. Once a design system is established it ensures brand and code consistency through its use. The best of these orchestrate a sort of harmony which can be noted across products and teams. This streamlines development efforts. The result is faster delivery which is focused on what makes a product great. By maintaining many of a website's core elements in a single source we can create quality modular components and patterns that benefit all.
