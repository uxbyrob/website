2.       Layout of the "additional / Extract Specific options":

 

 

File Preferences:

 

§  File Preferences include: the default setting is in bold

§  Level of Detail: (drop-down selector with 2, 4, 6, Most as options)

·         MOST - all HS codes at the most granular level (including country based codes of 8, 10, 12 digits - all terminal leaf nodes.

·         6 digit - all 2 and 4 digit codes at the 6 digit level and anything at the 6 digit level

·         4 digit - all 4 digit level codes for anything at the 2 digit level PLUS anything at the 4 digit level

·         2 digit - any selected code at the 2 digit level

§  ISO Country Code - two check boxes: Reporters and Trade Partners

·         Default is neither checked

·         This adds ISO codes to the output file

§  Field Separator: select only one: Comma Delimited, Tab Delimited

·         Default is Comma (CSV)

§  Text Qualifier: select only one: " or none

·         Default is "

§  Data Layout: select only one: Wide or Narrow

·         Default is Narrow

§  Prepare Data Extrace

 

Recurrence (yes/no). If yes, then:

 

 

·         user has the option to add recurrence to a saved data extract job.

·         Recurrence options:

§  Weekly

·         User can choose "Recur every  [XXX]  week(s) on: [YYY]"

·         where XXX is number between 1 and 99

·         YYY is picker for days of week

·         Job will start at COB (8 PM ET) on the selected day

·         Put note below the day selection (Data Extract will run overnight on the selected day

§  Monthly:

·         User can choose "Recur on the [XXX] day of every YYY Month(s)"

·         XXX - number 1-30

·         add remark: "For months with fewer than 30 days, data extract report will run on the last day of the month."

·         YYY - number from 1 to 99

 

Option to include email alerts notifications

·         Applicable For Saved Data Extracts,

·         Part of saving process includes email alert preferences

§  User can request immediate alert, when the job has finished processing

·         there is a simple control (yes/no) for the Email Alert which is part of the extract creation process

·         Default is "Yes, send immediate email alert"

§  Immediate Alerting subject to Connect Platform rules around embargos etc.

·         Email Alert can only be sent on accounts w/ associated email addresses (i.e. individual user accounts - no alerting for shared/shared IP account users) - for users without email address this option should not be shown
