How it works

An Alert is presented as a horizontal banner visually offset from the surrounding elements. The alert will in most cases use a contextual class to give it more meaning. The type in the alert may vary in weights and contain links.

 

With Alert background coloring, reds should only be used with warnings, such as when data loss is possible or a notable change in the system is imminent. Greens should indicate user-action success. Blues is used for information. Orange is used for grab attention for non-critical actions.

 

 

Where to use it

The Alert is used when usability principles dictate the system should communicate a current or impending state change to the user. An alert is a communicative element added to the page on a temporary basis.

 

Typical uses are to notify of system outages for maintenance, or other information broadly useful to users, to warn of an impending change that will significantly affect the user/their/data/their interactions on the system, or to communicate that a user action is complete, successful, or was not successful.

 

An Alert should appear near the top of the visual hierarchy, at the top of the element it describes, indicates, or warns about.

 

It should be brief and communicate in clear, understandable, non-jargon language. An Alert should provide follow up information or next steps if possible or applicable.

 

The default presentation of an Alert allows for its dismissal, the user-dictated removal from the interface. In cases where designers, stakeholders, or system administrators deem appropriate, the Alert can be non-dismissible, but this should be an exception, not the rule.
