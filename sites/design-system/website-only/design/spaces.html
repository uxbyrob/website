Spaces and hierarchy

 

Spaces help the user focus on the information they are looking for. By "boxing in" the eye we can reduce the cognitive load on the user by helping them eliminate distractions during in a task.

 

 

Design the negative space

A helpful hint in designing spaces— design the negative or white space around the content. White space can greatly improve the comprehension of a user. Variations in padding/margins can create large and small gutters and define a space.

 

Avoid sameness

Consistency in design is important to a polished look but variation it the key to usable design. Using a type hierarchy on an element level provides better readability.  Varying the padding between items helps define space. Mixing content types will also help break up the visual field.

 

Embrace the edges

Adding boxes inside of boxes and column just eats up white space. Let the design bleed from to the edge if a container or the browser window.

 

Page level Hierarchy

When starting a page layout, the most important step is creating an information hierarchy. Consider why the user is on the page and focus the design there first. Not all content on the page is equal.

 

Element level Hierarchy

Use type and spacing to create element level hierarchy. This also helps define the space.

 

Avoid double padding and double margins

 

Spaces can be defined with the restraint of boxes and lines. When we start creating spaces it's easy to default to putting groups in boxes but by stacking margins and padding around groups of boxes, we eat up all the white space that creates a space.
